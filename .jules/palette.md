## 2024-05-15 - Missing accessibility attributes on icon-only buttons
**Learning:** Found several icon-only buttons in the application using `TouchableOpacity` (e.g. `closeIconBtn` in `DrawerMenu`, `closeBtn` in `ProfileSwitcherModal`) that are lacking proper accessibility labels (`accessibilityLabel`), roles (`accessibilityRole="button"`) and hit slops. Without these, screen readers will not be able to convey the action of the buttons effectively to visually impaired users, reducing the overall accessibility of the app.
**Action:** Always verify that `TouchableOpacity` elements wrapping purely graphical components like `MaterialIcons` contain descriptive `accessibilityLabel` properties and proper `accessibilityRole` to support screen readers, and add `hitSlop` to make them easier to tap.
