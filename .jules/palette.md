## 2026-05-04 - Adding hitSlop and Accessibility to Modal Close Buttons
**Learning:** Found multiple instances of icon-only close buttons in modals (e.g. `ProfileSwitcherModal.tsx`, `ShiftTimeline.tsx`) lacking proper accessibility roles, labels, and an expanded touch target (`hitSlop`). React Native components like `TouchableOpacity` do not automatically inherit standard accessible properties from nested icon elements. Missing these attributes makes navigation difficult for screen readers and touch input error-prone.
**Action:** Always add `accessibilityRole="button"`, `accessibilityLabel={t('close')}` (or equivalent), and `hitSlop={{ top: 12, bottom: 12, left: 12, right: 12 }}` to standalone icon-only buttons to enhance accessibility and tap reliability. Ensure the target translation string exists in all locale files (e.g. `it`, `en`).
